## 一、核心代码

### 1. 手写 My_System_Init（HSI → HSE+PLL 72MHz）

```c
int My_System_Init(void)
{
    volatile uint32_t TimeOut;

    // ① 适配 Flash：72MHz 需要 2 个等待周期 + 预取缓冲
    FLASH_PrefetchBufferCmd(FLASH_PrefetchBuffer_Enable, ENABLE);
    FLASH_SetLatency(FLASH_Latency_2);

    // ② 开启 HSE（外部 8MHz 晶振），等 HSERDY
    RCC_HSEConfig(RCC_HSE_ON);
    TimeOut = 0x0500;
    while ((RCC_GetFlagStatus(RCC_FLAG_HSERDY) == RESET) && (--TimeOut));
    if (TimeOut == 0) { /* 错误处理：关 HSE+PLL，返回 */ }

    // ③ 配置 PLL：HSE × 9 = 72MHz
    RCC_PLLConfig(RCC_PLLSource_HSE_Div1, RCC_PLLMul_9);
    RCC_PLLCmd(ENABLE);
    TimeOut = 0x0500;
    while ((RCC_GetFlagStatus(RCC_FLAG_PLLRDY) == RESET) && (--TimeOut));
    if (TimeOut == 0) { /* 错误处理 */ }

    // ④ 配置总线分频：HCLK=72M, PCLK1=36M, PCLK2=72M
    RCC_HCLKConfig(RCC_SYSCLK_Div1);
    RCC_PCLK1Config(RCC_HCLK_Div2);
    RCC_PCLK2Config(RCC_HCLK_Div1);

    // ⑤ 切换系统时钟源为 PLL
    RCC_SYSCLKConfig(RCC_SYSCLKSource_PLLCLK);
    TimeOut = 0x0500;
    while ((RCC_GetSYSCLKSource() != 0x08) && (--TimeOut));
    if (TimeOut == 0) { /* 死锁，等看门狗复位 */ }

    return INIT_SYSTEM_SUCCESS;
}
```

> 在启动文件中注释掉默认的 `SystemInit`，改为调用此函数

### 3. 综合项目主流程

```c
void PROCESS(void)
{
    My_System_Init();                        // ① 系统时钟 72MHz
    NVIC_PriorityGroupConfig(NVIC_PriorityGroup_2);

    init_USART1();                           // ② 串口（中断接收）
    Process_I2C_OLED_Display();              // ③ I2C + OLED
    init_SPI();                              // ④ SPI + Flash
    init_PC13_GPIO();                        // ⑤ LED
    CE_Init_EXTI_For_Button();               // ⑥ 按键中断（保存/恢复）
    CE_Init_NVIC_For_USART();                // ⑦ 串口中断

    Maintain_Last_State();                   // ⑧ 从 Flash 恢复上次状态

    while (1)
    {
        // OLED 更新（标志位驱动）
        if (oled_update) {
            OLED_Show_LED_Status(oled_display_state);
            oled_update = 0;
        }

        // 左键：保存当前 LED 状态到 Flash
        if (flag_save) { /* 消抖 → SPI_W25Q64_Save_Byte() */ }

        // 右键：从 Flash 恢复 LED 状态
        if (flag_load) { /* 消抖 → SPI_W25Q64_Read_Byte() → 恢复 */ }

        // LED 状态机：常亮 / 常灭 / 闪烁
        if (led_mode == LED_MODE_ON)       LED_ON();
        else if (led_mode == LED_MODE_OFF) LED_OFF();
        else { /* 1000ms 周期闪烁 */ }
    }
}
```

### 4. 中断服务函数（标志位通信，不直接操作硬件）

```c
void USART1_IRQHandler(void)
{
    if (USART_GetFlagStatus(USART1, USART_FLAG_RXNE) == SET) {
        uint16_t ch = USART_ReceiveData(USART1);
        if (ch == '1')      { led_mode = LED_MODE_ON;   oled_update = 1; }
        else if (ch == '0') { led_mode = LED_MODE_OFF;  oled_update = 1; }
        else if (ch == '2') { led_mode = LED_MODE_BLINK; }
    }
}

void EXTI1_IRQHandler(void) { flag_save = 1; EXTI_ClearFlag(EXTI_Line1); }
void EXTI2_IRQHandler(void) { flag_load = 1; EXTI_ClearFlag(EXTI_Line2); }
```

## 二、核心知识点

### 1. 时钟树切换流程

```
上电默认 HSI(8MHz)
    ↓ 设 Flash Latency=2 + 预取缓冲
开启 HSE(8MHz)，等 HSERDY
    ↓ PLL = HSE × 9
开启 PLL，等 PLLRDY
    ↓ 设 AHB/APB1/APB2 分频
SYSCLK 切到 PLL → 72MHz
```

### 2. Flash 等待周期

| SYSCLK | Latency | 原因 |
|--------|---------|------|
| 0~24MHz | 0 | Flash 跟得上 |
| 24~48MHz | 1 | 需插入 1 个等待周期 |
| **48~72MHz** | **2** | 需插入 2 个等待周期 |

> 忘记设 Latency = 初学"超频死机"头号原因

### 3. 软件循环超时 + volatile

```c
volatile uint32_t TimeOut = 0x0500;
while ((条件 == RESET) && (--TimeOut));
```

- `volatile` 防止编译器优化掉循环变量
- `0x0500` ≈ 1280 次循环 ≈ 1~1.6ms，覆盖晶振起振时间
- 此时无定时器可用，只能用空循环估算

### 4. SysTick 模运算 — 溢出为什么不用怕

```c
uint32_t now  = GetTick();     // e.g. 0xFFFFFFF0
uint32_t prev = 0xFFFFFFF0;
// ... 计数器溢出归零，now = 0x00000010
uint32_t diff = now - prev;    // 0x00000010 - 0xFFFFFFF0 = 0x00000020 ✓
```

`uint32_t` 减法天然模 2³²，即使溢出差值依然正确——所有 `millis()` 类函数的数学基础。

### 5. 回调函数模式

```
库定义函数指针  →  用户写适配函数  →  注册指针  →  库按需回调
```

核心价值：库不绑定硬件，换 MCU 只需重写适配函数。

### 6. 综合项目架构原则

| 原则 | 实现 |
|------|------|
| 非阻塞主循环 | 状态机 + `GetTick()` 差值比较 |
| ISR 只设标志位 | 不直接操作硬件，主循环统一处理 |
| 中断优先级 | 按键(抢占0) > 串口(抢占1) |
| 状态持久化 | Flash 保存→上电恢复 |
| 多源控制解耦 | `led_mode` 状态变量统一仲裁 |
