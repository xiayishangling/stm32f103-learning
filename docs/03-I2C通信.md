## 一、核心代码

### 1. 初始化

```c
void init_I2C1(void)
{
    // I2C1 挂 APB1（32MHz），重映射至 PB8(SCL)/PB9(SDA)
    RCC_APB2PeriphClockCmd(RCC_APB2Periph_AFIO, ENABLE);
    GPIO_PinRemapConfig(GPIO_Remap_I2C1, ENABLE);

    RCC_APB2PeriphClockCmd(RCC_APB2Periph_GPIOB, ENABLE);
    GPIO_InitTypeDef gpio = {0};
    gpio.GPIO_Pin   = GPIO_Pin_8 | GPIO_Pin_9;
    gpio.GPIO_Speed = GPIO_Speed_50MHz;   // 400kHz 留足裕量
    gpio.GPIO_Mode  = GPIO_Mode_AF_OD;    // 复用开漏
    GPIO_Init(GPIOB, &gpio);

    // I2C 外设初始化
    RCC_APB1PeriphClockCmd(RCC_APB1Periph_I2C1, ENABLE);
    RCC_APB1PeriphResetCmd(RCC_APB1Periph_I2C1, ENABLE);  // 复位清除上次状态
    RCC_APB1PeriphResetCmd(RCC_APB1Periph_I2C1, DISABLE);

    I2C_InitTypeDef i2c = {0};
    i2c.I2C_DutyCycle  = I2C_DutyCycle_2;   // 2:1 占空比
    i2c.I2C_Mode       = I2C_Mode_I2C;
    i2c.I2C_ClockSpeed = 400000;            // 400kHz 快速模式
    I2C_Init(I2C1, &i2c);
    I2C_Cmd(I2C1, ENABLE);
}
```

### 2. 主机发送（精简版，核心流程）

```c
int I2C_Send_Bytes(I2C_TypeDef *I2Cx, uint8_t ADDr, uint8_t *pData,
                    uint32_t Size, int32_t TimeOut)
{
    if (pData == NULL || Size <= 0) return ERR;

    // 1. 等总线空闲
    uint32_t t = GetTick();
    while (I2C_GetFlagStatus(I2Cx, I2C_FLAG_BUSY) == SET) { /* 超时判断 */ }

    // 2. 发 START，等 SB
    I2C_GenerateSTART(I2Cx, ENABLE);
    t = GetTick();
    while (I2C_GetFlagStatus(I2Cx, I2C_FLAG_SB) == RESET) { /* 超时判断 */ }

    // 3. 清 AF（必须在 SB 之后、发地址之前！）
    I2C_ClearFlag(I2Cx, I2C_FLAG_AF);

    // 4. 发从机地址（写方向：末位 0）
    I2C_SendData(I2Cx, ADDr & 0xFE);

    // 5. 等 ADDR，同时监测 AF
    t = GetTick();
    while (1) {
        if (I2C_GetFlagStatus(I2Cx, I2C_FLAG_AF) == SET)
            { I2C_GenerateSTOP(I2Cx, ENABLE); return AF_ERR; }
        if (I2C_GetFlagStatus(I2Cx, I2C_FLAG_ADDR) == SET) break;
        /* 超时判断 */
    }

    // 6. 清 ADDR（读 SR1 再读 SR2）
    I2C_ReadRegister(I2Cx, I2C_Register_SR1);
    I2C_ReadRegister(I2Cx, I2C_Register_SR2);

    // 7. 循环发数据
    for (uint16_t i = 0; i < Size; i++) {
        t = GetTick();
        while (I2C_GetFlagStatus(I2Cx, I2C_FLAG_TXE) == RESET) {
            if (I2C_GetFlagStatus(I2Cx, I2C_FLAG_AF) == SET)
                { I2C_GenerateSTOP(I2Cx, ENABLE); return AF_ERR; }
            /* 超时判断 */
        }
        I2C_SendData(I2Cx, pData[i]);
    }

    // 8. 最后一字节后等 BTF + AF
    t = GetTick();
    while (1) {
        if (I2C_GetFlagStatus(I2Cx, I2C_FLAG_AF) == SET)
            { I2C_GenerateSTOP(I2Cx, ENABLE); return AF_ERR; }
        if (I2C_GetFlagStatus(I2Cx, I2C_FLAG_BTF) == SET) break;
        /* 超时判断 */
    }

    I2C_GenerateSTOP(I2Cx, ENABLE);
    return I2C_SEND_SUCCESS;
}
```

### 3. 主机接收（ACK/NAK 预约机制）

```c
int I2C_Recv_Bytes(I2C_TypeDef *I2Cx, uint8_t ADDr, uint8_t *pBuffer,
                    uint32_t Size, int32_t TimeOut)
{
    // ... 发 START → 清 AF → 发 ADDr|0x01 → 等 ADDR+AF → 清 ADDR ...

    if (Size == 1) {
        // 清 ADDR 后立刻 NAK+STOP → 等 RXNE → 读取
        I2C_AcknowledgeConfig(I2Cx, DISABLE);
        I2C_GenerateSTOP(I2Cx, ENABLE);
        while (I2C_GetFlagStatus(I2Cx, I2C_FLAG_RXNE) == RESET);
        pBuffer[0] = I2C_ReceiveData(I2Cx);
    }
    else if (Size == 2) {
        // ACK → 读第1字节 → NAK+STOP → 读第2字节
        I2C_AcknowledgeConfig(I2Cx, ENABLE);
        while (I2C_GetFlagStatus(I2Cx, I2C_FLAG_RXNE) == RESET);
        pBuffer[0] = I2C_ReceiveData(I2Cx);
        I2C_AcknowledgeConfig(I2Cx, DISABLE);
        I2C_GenerateSTOP(I2Cx, ENABLE);
        while (I2C_GetFlagStatus(I2Cx, I2C_FLAG_RXNE) == RESET);
        pBuffer[1] = I2C_ReceiveData(I2Cx);
    }
    else {
        // 前 Size-2 字节 → ACK 循环收
        I2C_AcknowledgeConfig(I2Cx, ENABLE);
        for (uint16_t i = 0; i < Size - 2; i++) {
            while (I2C_GetFlagStatus(I2Cx, I2C_FLAG_RXNE) == RESET);
            pBuffer[i] = I2C_ReceiveData(I2Cx);
        }
        // 倒数第二字节 → ACK → 读
        while (I2C_GetFlagStatus(I2Cx, I2C_FLAG_RXNE) == RESET);
        pBuffer[Size-2] = I2C_ReceiveData(I2Cx);
        // 最后字节 → NAK+STOP → 读
        I2C_AcknowledgeConfig(I2Cx, DISABLE);
        I2C_GenerateSTOP(I2Cx, ENABLE);
        while (I2C_GetFlagStatus(I2Cx, I2C_FLAG_RXNE) == RESET);
        pBuffer[Size-1] = I2C_ReceiveData(I2Cx);
    }
    return I2C_RECV_SUCCESS;
}
```

## 二、核心知识点

### 1. I2C 帧格式

```
[START] [7位地址 + R/W] [ACK] [8位数据] [ACK] ... [NAK] [STOP]
  SDA↓    从机地址       从机回  数据字节    从机回                     SDA↑
```

- **START**：SCL 高时 SDA 下降沿
- **STOP**：SCL 高时 SDA 上升沿
- 每字节后必须跟 1 位 ACK/NAK

### 2. 标志位速查

| 标志 | 含义 | 清除方式 |
|------|------|----------|
| **SB** | START 已发送 | 读 SR1 后写 DR 自动清 |
| **ADDR** | 寻址成功 | **读 SR1 再读 SR2** 自动清 |
| **AF** | 应答失败（粘性） | **手动** `I2C_ClearFlag()` |
| **TXE** | 发送寄存器空 | 写 DR 自动清 |
| **RXNE** | 接收寄存器非空 | 读 DR 自动清 |
| **BTF** | 字节发送完成 | 读 SR1 后写/读 DR |

### 3. AF 清除时机（最易出错的地方！）

```
发 START → 等 SB → 【此时清 AF】 → 发地址 → 等 ADDR
```

- AF 只在完整字节发送后（含 ACK 时钟）才可能置位
- 起始位阶段硬件绝不会改 AF，SB 后清 AF 不会漏判
- 在 SB 之前清 AF 没用（可能被残留 AF 干扰）

### 4. 接收的 ACK/NAK 预约机制

> NAK 和 STOP 是**预约指令**，不是事后反馈！

| 接收字节数 | 策略 |
|------------|------|
| **1 字节** | 清 ADDR 后立刻 `NAK + STOP` |
| **2 字节** | 清 ADDR 后 `ACK` → 收第1字节 → `NAK + STOP` → 收第2字节 |
| **>2 字节** | 前 N-2 字节 `ACK` 循环收 → 倒数第2字节 `NAK + STOP` → 收最后字节 |

### 5. 超时设计：单步超时 vs 总超时

| 场景 | 适用方式 | 原因 |
|------|----------|------|
| **I2C** | 单步超时（每步独立计时） | 从机可能拉长时钟，总线可能被占用 |
| **USART** | 总超时 | 串口通信时间可预测 |

### 6. 运算符陷阱

| 错误 | 正确 | 原因 |
|------|------|------|
| `ADDr || 0x01` | `ADDr | 0x01` | `||` 是逻辑或，结果只有 0/1 |
| `ADDr && 0xFE` | `ADDr & 0xFE` | `&&` 同理 |
| 地址参数用指针 | 地址参数用 `uint8_t` | 地址是单值不是数组 |

### 7. 防御性编程要点

- 超时/AF 时必须发 **STOP** 释放总线，并**清除 AF** 标志
- `GetTick()` 在循环外获取一次，循环内只做差值比较
- 初始化时先 `ResetCmd(ENABLE)` 再 `ResetCmd(DISABLE)` 复位总线状态
- GPIO 复用开漏仍必须配置 Speed，推荐 50MHz

## 三、心得

- I2C 比 USART 复杂一个量级：多从机、ACK 握手、粘性标志、NAK 预约——需要**严格按状态机顺序**编程
- AF 标志的清除时机是整个 I2C 驱动最容易出错的地方：SB 之后、发地址之前
- 接收多字节时 ACK/NAK 必须**提前一轮**设置——这是理解 I2C 的"顿悟时刻"
