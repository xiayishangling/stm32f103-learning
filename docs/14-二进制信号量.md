## 一、核心代码

### 1. ADC 注入序列状态机（RTOS 版）

```c
PS_IJ_State_Machine ps_state = PS_Init;

void vPS_state_machine_IS(void *argument)
{
    for (;;) {
        switch (ps_state) {
        case PS_Init:
            HAL_TIM_Base_Start(&htim1);          // TIM1 TRGO 启动
            ps_state = PS_Wait_ADC_JEOC;
            break;

        case PS_Wait_ADC_JEOC:
            HAL_ADCEx_InjectedStart_IT(&hadc1);  // 开启注入中断
            osStatus_t st;
            do {
                st = osSemaphoreAcquire(ADC_JEOC_BinarySemHandle, 100);
                if (st == osErrorTimeout && (ps_state == PS_Restart || ps_state == PS_Init))
                    break;
            } while (st != osOK);

            if (st == osOK) ps_state = PS_Process;
            break;

        case PS_Process:
            uint32_t jdr = HAL_ADCEx_InjectedGetValue(&hadc1, ADC_INJECTED_RANK_1);
            HAL_ADCEx_InjectedStop_IT(&hadc1);
            float voltage = jdr * (3.3f / 4095);
            vVoltage_to_LED(voltage);            // 通过队列发给 LED 任务
            // 串口打印电压
            ps_state = PS_Wait_ADC_JEOC;
            break;

        case PS_Restart:
            HAL_TIM_Base_Stop(&htim1);
            HAL_ADCEx_InjectedStop_IT(&hadc1);
            LED_OFF();
            osDelay(5000);                       // 5s 后自动重启
            ps_state = PS_Init;
            break;
        }
    }
}
```

### 2. 中断 → 信号量 → 任务

```c
// JEOC 中断回调 → 释放信号量
void HAL_ADCEx_InjectedConvCpltCallback(ADC_HandleTypeDef *hadc)
{
    if (hadc->Instance == ADC1)
        osSemaphoreRelease(ADC_JEOC_BinarySemHandle);
}

// 按键中断回调 → 释放信号量
void HAL_GPIO_EXTI_Callback(uint16_t GPIO_Pin)
{
    if (GPIO_Pin == PA3_KEY1_Pin && KEY_PRESSED())
        osSemaphoreRelease(KEY1_BinarySemHandle);
}

// 按键任务：获取信号量 → 消抖确认 → 重启 ADC
void vPS_KEY1(void *argument)
{
    for (;;) {
        if (osSemaphoreAcquire(KEY1_BinarySemHandle, osWaitForever) == osOK) {
            if (IS_KEY1_Click()) {
                osDelay(100);
                HAL_TIM_Base_Stop(&htim1);
                HAL_ADCEx_InjectedStop_IT(&hadc1);
                ps_state = PS_Restart;
            }
        }
    }
}
```

### 3. 电压 → 队列 → LED 任务

```c
void vVoltage_to_LED(float voltage)
{
    LED_Message msg;
    msg.state = (voltage > 1.5f) ? LED_State_ON : LED_State_OFF;
    osMessageQueuePut(LEDQueueHandle, &msg, 0, pdMS_TO_TICKS(100));
}

void vPS_LED_state_machine(void *argument)
{
    for (;;) {
        LED_Message msg;
        if (osMessageQueueGet(LEDQueueHandle, &msg, 0, pdMS_TO_TICKS(100)) == osOK) {
            switch (msg.state) {
                case LED_State_ON:  LED_ON();  break;
                case LED_State_OFF: LED_OFF(); break;
            }
        }
    }
}
```

## 二、核心知识点

### 1. 二进制信号量本质

```
二进制信号量 = 长度为1、不承载数据的特殊队列
只有两种状态：0（无信号） / 1（有信号）
不传数据，只传「事件是否发生」
```

| 操作 | API | 行为 |
|------|-----|------|
| **Give / Release** | `osSemaphoreRelease` | 置 1，唤醒等待任务 |
| **Take / Acquire** | `osSemaphoreAcquire` | 为 1 取走（置 0），为 0 则阻塞 |

> **重复 Give 不累加**：信号量已为 1 再释放，值保持为 1

### 2. 信号量三大应用场景

| 场景 | 模式 |
|------|------|
| 任务间同步 | 生产者完成任务 → Give → 消费者 Take 后继续 |
| 中断延后处理 | ISR 只 Give，业务逻辑在任务中 |
| 一次性事件标志 | 替代轮询，节省 CPU |

### 3. 中断→信号量→任务 vs 中断→队列→任务

| 方式 | 传数据 | 开销 | 适用 |
|------|--------|------|------|
| 队列 | ✅ 可传 | 较大 | 需要传数据的场景 |
| **信号量** | ❌ 不传 | **极小** | 纯同步/事件通知 |

### 4. 裸机 ADC 状态机 → RTOS 版对比

```
裸机版：
  主循环轮询 JEOC 标志 → 处理 → 间隔

RTOS 版：
  ADC 中断 → osSemaphoreRelease → 信号量唤醒 ADC 任务
  osSemaphoreAcquire 阻塞等待，不占 CPU
```

> 裸机主循环需要 `switch-case` + `GetTick()` 手动管理阻塞，RTOS 直接 `osSemaphoreAcquire` 一行搞定。

### 5. 踩坑记录

| 问题 | 原因 | 解决 |
|------|------|------|
| ADC 不进 JEOC 中断 | 用了普通轮询函数，没开中断 | 改用 `HAL_ADCEx_InjectedStart_IT` |
| 按键无法重启 | 接线接错引脚 | 检查 PA3 物理连接 |
| `ps_state` 超时跳出失败 | 超时后未检查是否是 Restart 状态 | 加 `ps_state == PS_Restart` 判断 |

## 三、心得

- 二进制信号量是 FreeRTOS 最轻量的同步机制——无数据、无所有权、无累加，纯粹的事件通知
- 中断→信号量→任务是裸机中断→标志位→主循环的 RTOS 版升级，思路一致但写法更优雅
- 将裸机的 ADC 注入序列状态机迁移到 RTOS，`osSemaphoreAcquire` 替代了 `while(currentTick - start < timeout)` 的忙等，CPU 彻底解放
